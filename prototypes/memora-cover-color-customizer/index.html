Tratamientos

Qué encontré en el repo

  • Las portadas/contraportadas vienen como location assets, no como fondos CSS editables.
  • El review usa ReviewBookFlipbookFaceContent: front con CoverYearOverlay, back con BackCoverSubtitle.
  • El admin ya tiene patrón de edición: color, weight, letter spacing, guías, recents y preview en vivo.

Regla UX recomendada

  • Usuario final: presets + tinte + color de texto.
  • Admin: editor completo de posición/guías.
  • PDF/review: misma data, cero divergencia visual.

Modelo de datos

Añadir estilo user-facing separado o extender CoverYearStyle con tintColor, tintOpacity y scope por cara. Mantener fallback por palette.

Componentes

Reusar SegmentedControl, ActionButton, SurfaceCard y controles de color del editor admin. No crear otro zoológico visual.

Accesibilidad

Contraste mínimo AA para overlays, labels reales, targets 44px en mobile y botón “usar color legible” cuando el picker falla.
